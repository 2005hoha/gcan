## 一、相关文献贡献总结

### 1.1 值分解方法演进（信用分配主线）

```
VDN (2017) → QMIX (2018) → QTRAN (2019) → QPLEX/WQMIX (2020-21) → DAGMIX (2023) → GCAN (ours)
   和            单调            非单调          加权/对偶          动态图             物理图
```

| 方法 | 年份 | 贡献 | 核心局限 |
|------|------|------|---------|
| **VDN** | 2017 | 提出 Q_tot = Σ Q_i，开创值分解范式 | 过于简单，无法表达 agent 交互 |
| **QMIX** | 2018 | 超网络生成非负权重，保证单调性 | 单调假设强约束，N>5 时限制性能 |
| **QTRAN** | 2019 | 放宽单调约束，引入 V_opt | 训练不稳定，实践中收敛困难 |
| **QPLEX** | 2021 | 双路架构，分离优势函数和值函数 | 架构复杂，参数冗余 |
| **WQMIX** | 2020 | 加权 QMIX，最优动作高权重 | 仅缓解，未解决单调约束问题 |
| **RES-QMIX** | 2021 | 正则化 + target_update=800 for MPE | 仅 MPE 特定修复，无结构创新 |
| **DAGMIX** | 2023 | 动态稀疏图 + Gumbel-Softmax 边选择 | 图在隐空间构建，不可解释，不传物理 |

### 1.2 GNN+MADRL 方法（图构建主线）

| 方法 | 年份 | 图类型 | GNN位置 | 测试环境 | 局限 |
|------|------|--------|---------|---------|------|
| **DGN** | ICLR'20 | 邻近图 | Agent层(编码) | SMAC | GNN仅做特征提取 |
| **G2ANet** | AAAI'20 | 动态注意力 | Agent层 | SMAC/MPE | Hard+Soft两阶段注意 |
| **Qedgix** | '24 | 距离图 | Agent层(编码) | UAV仿真 | EdgeConv不暴露注意力 |
| **DVD** | ICML'22 | 隐状态语义图 | Mixer层(因果) | SMAC/MACO | 语义图非物理，不跨sim2real |
| **QGNN** | '23 | 通信图 | Mixer层(轻量) | SMAC | EdgeConv，无注意力可解释 |
| **XMIX** | '25 | 全连接GIN | Mixer层(超网络) | SMAC/SMACv2 | Transformer+GIN复杂 |
| **GCAN** | ours | **物理空间全连接图** | **Mixer层(信用)** | MPE→VMAS | 开发中 |

### 1.3 LLM+MADRL 方法（LLM调度主线）

| 方法 | 年份 | LLM角色 | LLM频率 | 局限 |
|------|------|---------|---------|------|
| **RALLY** | 2024 | 高层目标规划 | 固定50步 | 拍脑袋定的频率，无理论依据 |
| **L2M2** | 2024 | 一次性子任务分配 | 1次/Episode | 离线，无在线适应性 |
| **CRAFT** | 2024 | 课程生成 | 完全离线 | 放弃在线信息 |
| **ReAd** | 2024 | 每步反思规划 | 每步 | 延迟爆炸，不可物理部署 |
| **YOLO-MARL** | 2025 | 奖励函数生成 | 1次 | 适应性差，信号会衰减 |
| **CoMLRL** | 2024 | 联合训练 | 训练时 | GRPO不稳定 |
| **HTA-Lite** | ours | **事件驱动调度** | **按需** | 待实现 |

## 二、现有方法在我们四个问题上的处理情况

### 问题1：LLM延迟 vs 物理实时（T_LLM≈2-15s vs T_CTRL≈1-10ms）

| 方法 | 策略 | 缺陷 |
|------|------|------|
| RALLY | 固定50步调用一次 | 50是任意值。巡航期浪费，危险期等不到 |
| L2M2 | 整个Episode只用一次 | 策略在变，LLM信号不再匹配 |
| ReAd | 每步都调 | 延迟15s/步，不可能部署 |
| CRAFT | 完全离线 | 完全放弃在线适应 |
| **我们的HTA-Lite** | **事件触发：偏离/危险/超时/达成** | 需要调阈值 |

### 问题2：信用分配 O(N²) → 可扩展

| 方法 | 策略 | 缺陷 |
|------|------|------|
| QMIX | 超网络 monotonic weights | 单调假设在N≥5失效（已观测到崩溃） |
| LCA | LLM pairwise O(N²) | 10 agent = 100次LLM调用 |
| RALLY RMIX | N→K=3角色压缩 | 3个角色表达不了复杂协作 |
| DVD | GAT做因果去混杂 | 图在隐空间，不是物理空间 |
| QGNN | EdgeConv通信图 | 无注意力可解释 |
| DAGMIX | Gumbel-Softmax动态图 | 边选择不可解释 |
| **我们的GCAN** | **GAT物理图→Softmax信用** | 需要证明优于VDN等baseline |

### 问题3：LLM信号衰减（时序错配）

| 方法 | 策略 | 缺陷 |
|------|------|------|
| YOLO-MARL | 训前生成，不再更新 | 信号明确会衰减（已分析三种模式） |
| RALLY | 固定角色定义不动 | 无法适应策略演化 |
| LCA | 一次性排序 | 无更新机制 |
| **我们的CoEvo-Lite** | **在线监测方差/熵/基尼系数** | 待实现 |

### 问题4：Sim2Real迁移

| 方法 | 策略 | 缺陷 |
|------|------|------|
| RALLY | 仅SITL截图 | 无真机部署 |
| CRAFT | GPS特权信息 | 真实环境无此信息 |
| VIKI-R | 视觉端到端 | 93%→28%崩溃 |
| DVD | 隐状态图 | 随训练漂移，不可迁移 |
| **我们的GCAN** | **物理坐标图** | 图拓扑sim/real近乎不变，是天然sim2real桥 |

### 3.3 跨框架 Reward 标准化公式

**MPE SimpleSpread 每步 Reward（源码验证）：**

```python
# PettingZoo simple_spread.py (已验证本地 conda 环境)
def reward(agent, world):
    rew = 0
    if agent.collide:
        for a in world.agents:
            rew -= 1.0 * (is_collision(a, agent) and a != agent)
    return rew  # 局部：每碰一对 = -1

def global_reward(world):
    rew = 0
    for lm in world.landmarks:
        dists = [||a.pos - lm.pos|| for a in world.agents]
        rew -= min(dists)  # 全局：每个地标 = -最近距离
    return rew

# SimpleEnv 混合公式：
# 每步 per-agent reward = local_ratio * local_reward + (1-local_ratio) * global_reward
# 默认 local_ratio = 0.5
```

**跨框架归一化因子：**

| 因子 | MARLlib | EPyMARL | 比率 |
|------|---------|---------|------|
| max_cycles (T) | 25 | 100 | ×4 |
| N agents/landmarks | 3 | 5 | ×~1.67 per step |
| 每步 global_reward 期望 | −3×0.6≈−1.8 | −5×0.5≈−2.5 | ×1.39 |

**归一化公式：**

```
R_normalized(N5, T100) ≈ R_MARLlib(N3, T25) × (T_ours/T_them) × (N_ours/N_them)
                        ≈ R_MARLlib × 4 × 1.39
                        ≈ R_MARLlib × 5.56
```

### 3.4 归一化后的可比分数

```
MARLlib QMIX N=3:  -183.8 → 归一化 N=5: ≈ -183.8 × 5.56 ≈ -1022
MARLlib VDN N=3:  -190.0 → 归一化 N=5: ≈ -190.0 × 5.56 ≈ -1056
我们的 GCAN N=5:   -532.6 (seed=42, 2M)
```

**GCAN 比归一化 QMIX 好 ~48%，比归一化 VDN 好 ~50%。**

## 四、我们的差异化贡献总结

### GCAN 的四个不可消解优势

1. **物理空间图 vs 语义图/隐状态图**
   DVD/DAGMIX 的图在 RNN 隐空间或学习到的空间，随训练漂移。GCAN 的图节点特征 =
   位置/速度/距离（物理量），图拓扑在仿真和真实中高度一致——天然 sim2real 桥。

2. **直接信用分配 vs 间接特征提取**
   Qedgix/QGNN/DGN 把 GNN 放 Agent 层做更好的特征编码，信用分配仍然靠 QMIX 超网络。
   GCAN 把 GAT 放在 Mixer 层——注意力权重就是信用分配，直接可解释。

3. **Softmax 信用 vs 单调权重**
   QMIX 的 abs(w) 保证 ∂Q_tot/∂Q_i ≥ 0——在 agent 互相掩护/牺牲的场景下是错的。
   GCAN 用 Softmax：允许 agent 信用互为消长（总和=1），自然表达替代/互补关系。

4. **O(1) LLM vs O(N²) LLM**
   LCA 的 pairwise 比较随 agent 数平方增长。GCAN 的 LLM 原型只调用 1 次（训练前），
   在线信用分配全由 GAT 做（毫秒级）。
