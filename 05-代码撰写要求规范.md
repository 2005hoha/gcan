## 二、代码风格规范

### 2.1 Python基础规范

- **行宽限制**：100字符
- **缩进**：4空格（不用Tab）
- **编码**：UTF-8
- **引号**：双引号用于docstring，单引号用于其他字符串
- **类型注解**：所有公开函数必须加类型注解

### 2.3 禁止事项

| 禁止 | 替代方案 |
|------|---------|
| `import *` | 显式导入需要的名称 |
| 魔法数字 | 定义为类常量或超参数 |
| `except:` 裸异常 | `except SpecificError as e:` |
| 超过4层的嵌套 | 提取为函数 |
| 函数超过50行 | 拆分为子函数 |
| `print()` 调试 | 使用 `logging` 模块 |
| 硬编码路径 | 使用 `pathlib.Path` |

## 四、训练流程规范

### 4.1 标准训练循环

```python
def train_gcan(config_path: str):
    """
    GCAN训练入口。
    
    流程：
    1. 加载配置
    2. 初始化环境（MPE → VMAS → PX4 SITL）
    3. 初始化组件（GraphBuilder, GCANMixer, Agent, Buffer, Learner）
    4. 训练循环
    5. 定期评估 + checkpoint保存
    """
    
    # 1. 配置
    config = load_yaml(config_path)
    
    # 2. 环境
    env = make_env(config.env)
    
    # 3. 组件
    graph_builder = GraphBuilder(config.graph)
    mixer = GCANMixer(config.mixer)
    agent = RNNAgent(env.obs_dim, config.agent)
    buffer = ReplayBuffer(config.buffer, env.n_agents)
    learner = GCANLearner(config.learner, mixer, agent)
    
    # 4. 训练
    for episode in range(config.n_episodes):
        batch = runner.run_episode(env, agent, mixer)
        buffer.insert(batch)
        
        if buffer.can_sample():
            sample = buffer.sample(config.batch_size)
            metrics = learner.train(sample, episode)
            
            # 日志
            if episode % config.log_interval == 0:
                wandb.log(metrics)
            
            # 评估
            if episode % config.eval_interval == 0:
                eval_metrics = evaluate(env, agent, mixer, config.n_eval_episodes)
                wandb.log({"eval/" + k: v for k, v in eval_metrics.items()})
            
            # Checkpoint
            if episode % config.save_interval == 0:
                save_checkpoint(agent, mixer, optimizer, episode, config.save_dir)
```

## 五、实验管理规范

### 5.1 可复现性要求

```python
# 每个训练脚本必须：
def set_seed(seed: int = 42):
    random.seed(seed)
    np.random.seed(seed)
    torch.manual_seed(seed)
    torch.cuda.manual_seed_all(seed)
    torch.backends.cudnn.deterministic = True
```

### 5.2 实验记录要求

每个实验必须记录：
- Git commit hash → `wandb.config.git_commit`
- 完整配置 → `wandb.config` (自动)
- 随机种子 → `wandb.config.seed`
- 硬件信息 → GPU型号、CUDA版本
- 训练时间 → wall clock time
- 所有超参数 → 配置文件全部上传

### 5.3 对比实验规范

当对比GCAN vs QMIX时：
```python
# 必须保证公平：
# - 相同的随机种子
# - 相同的环境参数
# - 相同的训练步数（2M for MPE）
# - 相同的评估协议（100 episodes, greedy policy）
# - 相同的网络容量（GCAN的参数总数 ≈ QMIX的参数总数）
```

### 7.2 代码注释原则

- **不要解释代码在做什么**（代码本身已经说明）
- **要解释为什么这样做**（设计决策、数学原理、边界条件）
- **标注已知局限性**

```python
# 使用Softplus而非ReLU：保证信用≥0（单调性约束），
# 同时在0附近平滑，不截断梯度。
# 已知局限：当所有信用都趋近0时，梯度消失。已通过归一化缓解。
credits = F.softplus(raw_credits)
```
